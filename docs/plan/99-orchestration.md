## 1. Objectif & ce que le plan prouve

Poser **l'assemblage** sur les six couches acquises : une **table des états** qui se dérive de règles de couplage (jamais une matrice figée), un **chemin d'un tour** où c'est *elle* qui décide d'agir (aucun trieur devant sa porte), un **accusé** qui est un filet et jamais un mensonge, un **prompt** composé identité → mémoire → cadre à budgets cloisonnés, une **UI témoin** qui ne stocke jamais rien, une **grille** minimale, un **gabarit fork** honnête sur l'agentique **et** sur la sécurité, une **échelle des mains** dont l'inconnu est gravé tel quel.

**Ce qu'il doit prouver** :
- Le **chemin d'un tour** est un couple **grille + elle**, **sans étage de classement intermédiaire** — bavardage et action passent par la **même** invocation (fidèle à l'âme : c'est elle qui décide d'agir, pas un trieur).
- L'**accusé ne ment jamais** : le minuteur ne joue que des phrases **vraies par construction** ; **aucune assertion d'acte** ne sort d'un filler (garde OF1).
- Le **prompt** tient l'ordre **identité entière d'abord**, **budgets cloisonnés** (persona jamais tronqué, aucun vase communicant) — et son **cadre porte les DEUX consignes** (tag d'humeur **et** signal de fermeture des `self_notes` — AF-9).
- **Tout affichage est une vue dérivée de l'état** : après crash/reboot/un mois, chaque surface reflète l'état source — **aucun composant d'assemblage ne stocke de contenu** (obligation, classe 2).
- **Aucune combinaison d'états hors des règles de couplage** n'est atteignable (le général par les règles, le délicat par les cas nommés).
- Le **fork obtient la maison, jamais l'habitante** ; le README ne survend **ni** l'agentique multi-provider **ni** la sécurité (posture nommée + modèle de menace, obligation conv 20).

**Ce que ce plan ne couvre PAS** : toute mécanique vit dans son plan d'origine — la voix (`plan/01`), la mémoire (`plan/02`), l'âme (`plan/03`), le proactif/tablée (`plan/04`), les organes vitaux (`plan/05`), le socle (`plan/00`). Ce plan **compose** ; le **détail visuel de l'UI**, le **fichier README** du fork et les **valeurs chiffrées** → rubrique 6.

- **Contenu** :

### O2 — L'accusé oral : le filet temporel + la garde d'honnêteté (99-B, OF1)
- **But** : l'accusé local pré-synthétisé joue **uniquement si le premier mot du cerveau n'arrive pas sous X ms** ; la **garde OF1** : le minuteur ne joue que des phrases **vraies par construction** ; toute **assertion d'acte** ne sort que du **stream réel** ou d'une **substitution TTS d'une phrase réellement produite**.
- **Dépend de** : `01` V13 (`cmd.tts.cache`, phrases pré-synthétisées) + V7 (stream TTS) · `05` R2 (accusés pré-synthétisés = usage JEU/substitution, OF1) · O1.
- **Fichiers** : `src/orchestrator/turn/ack/`.
- **Contenu** :
  - **Le filet** : l'accusé (cache `01` V13) joue **seulement si TTFT ≥ X ms** — le silence ne dépasse jamais X **par construction** (« le silence = panne perçue », cahier) ; un stream rapide ne déclenche **aucun tic**.
  - **La garde OF1** (deux usages du même cache, deux régimes) : le **filler d'attente au minuteur** = phrases **vraies par construction** (« je réfléchis », « deux secondes » — l'invocation est en vol, c'est un fait) ; **toute phrase qui affirme un acte** (« c'est noté », « je lance ça ») ne sort que de son **stream réel**, ou du cache **en substitution TTS d'une phrase que le cerveau a réellement produite** (l'usage JEU de `05` R2 : une économie de synthèse, **jamais un filler d'attente**). *(« "c'est noté" est vrai au moment où elle le dit » — `02` M3/§2.3 — descend jusqu'au cache audio.)*
  - **L'enforcement d'OF1 — pas une bonne intention (saveur classe 7)** : le **sous-ensemble « fillers d'attente » du cache `01` V13 est une liste FERMÉE et curée** (Phase 3, §6), **structurellement disjointe** des phrases d'acte. **⚠️ L'éligibilité au minuteur est un allow-list POSITIF (fail-closed), JAMAIS un filtre d'exclusion sur le cache commun (croisé conv 21, F-99-4 — MINEUR)** : une **nouvelle entrée** ajoutée au cache (ex. une phrase d'acte brève pour l'usage JEU) **n'est PAS éligible par défaut** — sinon le **fail-open que le fail-closed de B1 a proscrit** se rejoue (« tout le cache **sauf** {exclusions connues} » laisserait passer le neuf → une assertion d'acte jouée à l'attente = OF1 violé). **Seul ce sous-ensemble positif est éligible au déclencheur minuteur**, et une **assertion d'acte ne route JAMAIS depuis le minuteur** (uniquement stream réel ou substitution d'une phrase réellement produite). La garantie tient par **allow-list fermé positif + routage**, pas par la discipline — asserté au test (le jeu de fillers ne contient aucune assertion d'acte ; **une entrée de cache neuve, non taguée filler, n'est PAS tirée au minuteur** ; le minuteur ne tire que dans le jeu positif).
  - **Action longue** : sa **première phrase streamée est l'accusé riche** (« ok, je lance ça… ») — c'est elle qui parle. Les états de panne gardent leurs messages propres (`01` S11 · `05` R5 quota) — l'accusé **ne les remplace jamais**.
- **Fait quand** : TTFT < X → **aucun accusé** ; TTFT ≥ X → accusé **déclenché à X** ; **aucune phrase du chemin minuteur n'affirme un acte** (assertion OF1) ; une assertion d'acte ne sort que du stream réel / d'une substitution d'une phrase réellement produite ; le silence perçu ne dépasse jamais X (mesuré bout-en-bout). *(U-O2.)*
